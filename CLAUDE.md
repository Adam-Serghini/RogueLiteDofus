## Project state

**V0 + V1 are implemented** (TS + Vite, vanilla DOM). The original V0/V1 spec files have been removed from the repo — the implementation itself is now the reference for what exists. The only remaining doc is:

- `GDD-roguelite-dofus.md` — the full game vision. Background only; consult it for *why* a system exists, never to widen scope beyond the current tranche.

Implemented beyond the original specs (added by playtest iteration): a **4-element system** (`Element = terre | feu | eau | air`, `types.ts`) driven by 4 offensive stats (force→terre, intelligence→feu, agilité→air, **chance→eau**; the earlier wakfu/stasis elements were removed), **7 classes** (Iop/Cra/Eniripsa/Sadida/Sram/Feca/Ecaflip — **Sadida is disabled**: kept in `CLASSES`/data for tests & saves but filtered out of `classesDisponibles()` in `run.ts` pending a rework), a **recruitment system** (start with 2, recruit/heal at tavernes, team of 4 max), and **equipment** (`ITEMS`/`PANOPLIES` in `data.ts`; **4 slots** — `arme`/`coiffe`/`cape`/`anneau` (amulette/ceinture/bottes were removed to cut micro-management and leave room for future item rarity); each panoplie has 4 pieces with set bonuses at 2/4; **12 zone-bound panoplies** (one per zone, mapped in `BUTIN_ZONE`) — Aventurier, Paysan, Bouftou, Tofu, Gladiateur (Akadémie des Gobs), Kankreblath, Fantomatique (Maison Fantôme), Scarafeuille (defensive/all-element resist), Forgeron (force + prospection), Larve (vita+resists), Corailleur (chance/eau), **Kwak** (all-stat, best T1 set, Nid du Kwakwa); item stats are **rolled at drop** from `[min,max]` ranges (`Item.rolls` → an `ItemInstance` with concrete `stats` via `rollItem`), so two copies differ; duplicates allowed; **per-run** loot — `RunState.inventaire` + `PersoState.equipement` hold `ItemInstance` objects, lost on death; **Prospection** boosts drop rate). Equipment stats/PV/resistances fold into the combatant via `bonusEquipement`/`pvMaxPerso` in `run.ts`.

Also: **Archimonstres & Dofus Ocre** — any enemy can rarely spawn as a boosted "Archi" variant (`ARCHI` in `data.ts`); defeating it captures the species once (persistent `Meta.archis`), tracked in a per-zone **Bestiaire** (`showBestiaire` — lists **every** species of each zone incl. boss/miniboss, with capture state on the archi-capable ones; 32 capturable species across the 12 zones; the Kankreblath and Larves-donjon species genuinely have no archi on Metamob — archis attach to the base/outdoor variants). Zone rosters follow the **real DofusDB dungeon compositions** (`api.dofusdb.fr/dungeons/{id}` → `monsters[]` + `bosses[]`) — check there before inventing species. The **Dofus Ocre** is no longer a boss drop — it unlocks in tiers every 50 captures (`OCRE_PALIERS`), granting escalating team passives (PA + damage) via `bonusEquipe(meta)` (combines Dofus + Ocre). Not reachable with the current ~16 monsters by design (framework for a bigger bestiary, ~286).

Still out of scope (next tranches): kamas & economy nodes (Commerce/Forgemagie/HDV), multi-world forks, pet/multi-ring slots, item level requirements. (The weapon slot now exists; pet/multi-ring remain deferred.)

Both docs are in French; the game's domain vocabulary (PA, sorts, esquive, élément de frappe, Dofus…) is French and is kept as-is in code identifiers and data.

## Commands

- `npm run dev` — Vite dev server (hot reload) at http://localhost:5173
- `npm test` — run the Vitest suite once. `npm run test:watch` to watch.
- A single test: `npx vitest run src/combat.test.ts -t "règle de ligne"` (`-t` matches the `describe`/`it` name).
- `npm run typecheck` — `tsc --noEmit`, strict mode.
- `npm run build` — typecheck + production bundle to `dist/`.
- `npm run sim` — **headless balancing harness** (`src/sim.ts` via `vitest.sim.config.ts`; kept out of the `*.test.ts` glob so `npm test` ignores it). Replays every encounter N times (IA both camps, seeded RNG) for a reference multi-element team at each zone's expected level, in three gear states (**NU / MI 2-pièces / SET complet**), and prints a win% · turns · HP-left table (flags: `falaise 2→4p`, `stalemate?`…). Tune constants in `data.ts`/`progression.ts`, re-run, compare. **Caveat**: the sim's `controllerIA` heals optimally and never mis-plays → it's a difficulty *floor*; real play is harder (esp. boss fights). See the `sim-harness` memory.
- `node scripts/fetch-assets.mjs` — one-shot fetch of monster sprites from DofusDB (name.fr → gfxId → PNG into `public/assets/monstres/`; skips existing files; direct gfxId overrides for species without an exact DofusDB homonym).

## What the game is

A turn-based tactical roguelite in the Dofus universe, played in the browser. The single loop V0 must validate: **tactical combat + a permanent relic (Dofus) that survives death and makes the next run easier**. Everything is judged against one question — *is the combat fun?*

## Tech stack

- **TypeScript + Vite**, vanilla — **no React, no Canvas**. UI is plain DOM. The point is to test *feel*, not visuals.
- **Data-driven**: classes, spells, monsters, encounters, map-generation params and tuning constants live in `src/data.ts`, not hardcoded in logic.
- **Vitest** for the pure modules (`combat`, `progression`, `carte`) — validated headlessly, independent of the DOM.
- Persistence: the `Meta` object (owned Dofus, captured `archis`, `runs`/`victoires` counters) survives runs, saved to `localStorage` (key `rld_meta_v0`, with rétro-compat defaults in `chargerMeta`). Player *settings* (end-turn key, auto-end-turn, default formation row, per-class preferred element) persist separately under `rld_settings_v0` (`src/config.ts`). The **run in progress** persists under `rld_run_v0` (`sauverRunEnCours`/`chargerRunEnCours`/`effacerRunEnCours` in `run.ts`): saved after every resolved board node and at zone transitions; a combat in progress is NOT saved (refresh = redo the node); cleared on wipe/victory/abandon (abandon counts as a failed run); the start screen offers Reprendre/Abandonner when a save exists. Levels/points/HP live in `RunState` and reset on death.
- Deploy: pushing `main` publishes to GitHub Pages via `.github/workflows/deploy.yml` (`GITHUB_ACTIONS=true` sets Vite `base` to `/RogueLiteDofus/`).

## Code map & architecture

The hard line in this codebase: **`combat.ts` is pure** (no DOM, no `localStorage`) and **`ui.ts` is presentation only** (no game rules). Everything is wired in `main.ts`. Keep new logic on the correct side of that line — it's why the engine is unit-testable.

- `src/types.ts` — the data model (interfaces straight from the spec): combat types, `Progression`, equipment types (`Item` with `rolls`, the rolled `ItemInstance`, `EquipSlot`), and map types (`MapNode`, `GameMap`).
- `src/data.ts` — all content & tuning: `SORTS` (**7 classes**: Iop, Cra, Eniripsa, Sadida, **Sram**, **Feca**, **Ecaflip** + monsters), `CLASSES`, `MONSTRES` (bestiary across 12 zones), `DOFUS` (31-relic catalogue, only Pourpre has an effect), `COMBATS` (per-zone encounters; boss encounters = **boss + miniboss**), **`ZONES`** (12 zones, flat) + **`TRANCHES`** (level-bracket groupings; t1 = the 12 zone ids in play order — Incarnam → Champs d'Astrub → Tainéla → Tofus → Akadémie des Gobs → Kankreblath → Maison Fantôme → Scarafeuilles → Forgerons → Larves → Grotte Hesque → Nid du Kwakwa, ordered by official dungeon level; t2-t5 empty/inactive; event dungeons Nowel/Halouine/Pwak are reserved and must never enter tranche zone lists; each zone has `pools = {normales, elite, boss}`), `XP_PAR_TYPE`, `TAVERNE_PCT`, `GEN_CARTE`. Each class/monster has an optional `img` (path under `public/assets/…`). Sprites/icons and base stats/resistances are pulled from **DofusDB** (`api.dofusdb.fr` JSON + `/img/monsters/{gfxId}.png` CDN — keyed by **gfxId**, not monster id; the website itself is a JS SPA and not fetchable). **Archimonstre names** (`archiNom`) come from **Metamob**, which is more complete than DofusDB for archis — see the `archimonstre-source-metamob` memory for how to read its bundled static catalog without an API key.
- `src/combat.ts` — **pure engine**. `runCombat()` is the async turn loop, decoupled from input via **controllers** (`Controller = (acteur, combatants) => Action | null`). The UI passes a promise-based controller for the player; `controllerIA` drives enemies. RNG injected (`hooks.rng`) for deterministic tests. Reads final `c.stats` (via `statsEffectives`, which adds temp stat buffs; it never knows about levels except `c.niveau`, baked for `+x/lvl` scaling). Handles the **support + new-class mechanics** (see below). Dedicated handlers `lancerTarot`/`lancerEspritFelin` for the random Ecaflip spells.
- `src/progression.ts` — **pure**. Levels/XP/points: `xpRequis`, `coutPoint` (escalating 1→2→3 by points already invested), `gagnerXP`, `investir`/`investirN` (bulk allocation, `n = Infinity` for the UI's « Max » button), `restat`, `statsFinales`, `pvMaxFor`, `multOffensif`, `multSoin`, `pctRembPA` (PA-refund chance from Chance, used by Flèche magique). No DOM/storage. `STAT_KEYS` = investable stats (Soin & Prospection are class-only, not investable).
- `src/config.ts` — player settings (`Settings`): end-turn key, `autoFinTour`, default `formation` (classe → grid cell 0-7) and per-class preferred `elements`, persisted to `localStorage` (`rld_settings_v0`) with validation + defaults on load. `nouvelleRun` reads it to seed positions/elements.
- `src/carte.ts` — **pure**. `genererCarte(rng, pools)` builds one zone's node graph by rows with connectivity guarantees (combat ids drawn from the zone's `pools`); `atteignables(carte)` / `noeud(carte,id)` for navigation.
- `src/run.ts` — `RunState`/`PersoState` (per-character progression + current HP + `elementChoisi`, persisted across nodes). **Dynamic roster**: `nouvelleRun(choix)` builds the team from chosen class ids (run starts with **2 of 7**); `recruter`/`propositionsRecrutement`/`classesHorsEquipe`/`equipePleine` drive Taverne recruitment (team max 4, recruit-at-team-level, swap when full). `EQUIPE_DEPART` is just the default used by tests. Combatant factories **bake final stats** into a `Combatant` (`combattantDepuisPerso`, `equipeCombattante`, `synchroniserPV` writes back HP + chosen element), `soignerEquipe`, `Meta` persistence (`bonusDegatsDofus`).
- `src/ui.ts` — DOM rendering + player controller (click spell → click target) + screens: combat, **`showChoixEquipe`** (pick 2 of 7 at run start), **`showTaverne`** (heal OR recruit one of 2 random candidates; replace a member when full), start/transition/wipe/dofus, `showStatPanel` (level-up/Otomai), `showCarte` (SVG node graph), `showFormation`. Combatant cards show element ronds (clickable to switch strike element), crit/dégât-crit/soin/dgts-finaux mini-stats, all-4 resistances, prospection (allies only). Sprites/icons via the `asset(categorie, id)` convention (`/assets/<cat>/<id>.png`) with `onerror` fallback — drop a file, it shows; absent, it's removed (nothing breaks). Class portraits exist for all 7 classes under `public/assets/classes/`; individual spell icons may still be absent and fall back gracefully.
- `src/main.ts` — orchestration: accueil → for each zone of the **active `TRANCHES` entry** (une run = une tranche; t1 = 12 zones, t2-t5 locked placeholders shown grayed on the start screen), `jouerZone` generates that zone's board and navigates it until the donjon (boss+miniboss) is beaten → next zone; wipe anywhere ends the run. HP/levels persist across zones; only `Meta.dofus` survives death. The long-term content roadmap (5 tranches, rarity, boss signatures) lives in `PLAN-CONTENU.md`.

Two key seams keep things testable and decoupled:
1. **Combatants are rebuilt per fight** from `RunState` — final stats and `pvMax` are computed in `run.ts` and baked into the `Combatant`, so `combat.ts` stays ignorant of levels/points. HP persists via `PersoState.pvActuels` (written back by `synchroniserPV`), not by keeping the combatant object around.
2. **The async controller pattern**: `runCombat` `await`s the active camp's controller, so the same loop runs headless (IA vs IA in tests) or interactively (promise resolved by a DOM click) with no branching in the engine.

### Core combat rules that are easy to get wrong

- **Élément de frappe** (`elementDeFrappe`, `combat.ts`): a character has no fixed element. Element ↔ stat is `terre→force`, `feu→intelligence`, `air→agilité`, `eau→chance`. The strike element is the **player's `elementChoisi` if it is one of their 2 strongest elements, otherwise their single strongest** — so the clickable element ronds let a player pick between their top-two, not any element. All of a character's damage uses that element, and target resistances are applied per that element.
- **Grid & line rule (SYMMETRIC, 4×2 per camp)**: `position` is a grid cell **0-7** — cells **0-3 = ligne avant**, 4-7 = arrière (`estAvant(c) = position < 4`, `combat.ts`). `ennemi_ligne` spells hit only the target camp's front-row living; if the front row is empty, the back becomes exposed (`ligneFront`). Applies to **both** camps. `ennemi_tous` ignores it; rebound (Flèche perçante) hops by ascending position; taunt (`provoque`) overrides. Placement is free: all-front, all-back, mono-tank, etc. Players placed via **formation** (`config.formation` = `{classeId: cell}` → `PersoState.position`), edited from the board grid (`showFormation`); enemies placed by encounter `position` cells in `data.ts` (up to 8). Combat cards are vertical, grouped into front/back rows per camp.
- **Damage pipeline** (order matters, see `degatsCible` in `combat.ts`): esquive check (Agilité, capped) → roll (or max if `maxRoll` charge) → add élément-de-frappe stat × scaling → crit (Force, capped, *adds* damage via Agilité, does not double) → `degatsInfliges` debuff → rebound mult → resistance (unless `ignoreResistances`) → Dofus team bonus (`ctx.playerDamageBonus`, player camp only) → `multOffensif` (Intelligence, capped, all casters) → round, floor at 0.
- **Support mechanics** (Eniripsa kit, all in `combat.ts`): damage goes through `infligerDegats` → **shield (`bouclier`) absorbs before HP**; `poison`/`hot` are timed effects in `effets[]` that **tick at the start of the actor's turn** in `effetsDebutTour` (poison can kill, then transmits to the combatant `derriere` if `transmet`); heals via `soigner` (incl. `soinComplet`, `allie_tous`, `soinEquipeRatio` lifesteal-to-team); `appliquerSoutien` applies bouclier/HoT/`dissipe`/`paGain`/`bonusProchainSortPct`; per-target **cooldowns** live on `acteur.cooldowns` and are enforced inside `ciblesValides`. `allie_tous`/`soi`/`invocation` spells resolve with no target click.
- **Invocation & init (Sadida kit)**: the turn loop re-picks the next actor each step by **effective initiative** (`initOf` = base + `initiative` effects), so a mid-round init debuff (Déferlante/Étreinte) actually delays enemies that haven't acted yet. **Invocations** (`estInvocation`, e.g. the Poupée de garde via `invoquerPoupee`) occupy a slot, are skipped in the loop, and can `provoque` — enforced in `ciblesValides` (enemies must target a provoking ally first). One poupée per summoner. `Vigueur des bois` sets `bonusOffensifProchain`, a one-shot % consumed by the next damage cast.
- **Secondary-stat caps are mandatory**: crit, esquive, puissance offensive are capped (~50%) in the formulas. Don't remove the `Math.min` caps.
- **HP persists between fights/nodes** within a run — the Taverne node is the only heal. A wipe ends the run but **must not erase `Meta.dofus`**.
- **PV model**: `pvMax = pvBase + vitalitéFinale × PV_PAR_VITA` (spec-literal). Monster damage in `data.ts` is tuned against the resulting (inflated) player HP — retune both together.

## Scope discipline

Each tranche is deliberately narrow; the spec's validation criteria define "done". Current build: **6 playable classes** (Iop/Cra/Eniripsa/Sram/Feca/Ecaflip; Sadida disabled pending rework), **4 elements**, in-run levels/points, recruitment (start at 2, taverne recruit/heal, max 4), **equipment** (12 zone-bound panoplies, 4 slots, rolled-stat per-run loot), **Tranche 1 = 12 sequential zones** ordered by official dungeon level (see `TRANCHES` above; only Kardorim & Bouftou Royal drop a Dofus; all other bosses reserved), a 31-relic collection (only Dofus Pourpre has a gameplay effect so far), and a start screen showing the 5 tranches (t2-t5 locked). **All 12 bosses have a signature mechanic** on top of the shared `ecrasement/charge/morsure` kit. Damage-spell signatures: a 6-PA spell with a rider, listed FIRST in `sorts` (aggressive IA plays highest-cost first, ties broken by list order) and gated by `cooldownTours`. Engine-level signatures: Kwakwa's `mueElementaire` (each of its turns: 65% resist everywhere except one random element at 0 — `appliquerMueElementaire`), Grunob's `bonusParAllieLigne` passive (+15% damage per living same-row ally, folded into the damage mult in `lancerSort`), and invocation-type signatures played by `iaAgressif` in priority when `invocationUtile` (Kankreblath `invoqueMonstre` summons a zone monster — summons are FULL combatants that play their turns, unlike the Sadida Poupée; Shin Larve's 2-PA ponte lets her summon AND attack; Boostache `ressuscite` revives a fallen non-boss ally at 50% HP). Tests in `boss.test.ts`. Sim note: summoned actors inflate the sim's per-actor turn counts on those boss rows. Then item rarity. Everything else in the "out of scope" list above is **deferred** — resist pulling it in. **Balance is sim-tuned over the 12 zones (2 passes, NU/MI/SET columns), not yet playtested**: boss MI (2-piece) win% now ramps ~47→59→42→78→45→56→70→89→89→95→50→5 with the Kwakwa finale at 77% full-set (sim = difficulty floor). Known remaining issues: Tofus & Maison Fantôme elites are near-impossible *naked only* (fine mid-tranche, MI 100%), late-boss MI still cushy, and geared normals stay trivial — the future rarity system is expected to smooth the gear curve before another tuning pass. Values in `data.ts`/`progression.ts` are meant to be adjusted. Crit is a multiplicative bonus (affects players AND monsters).
