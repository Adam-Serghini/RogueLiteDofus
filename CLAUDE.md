### Archimonstres & Dofus Ocre

Any enemy can rarely spawn as a boosted "Archi" variant (`ARCHI` in `data.ts`); defeating it captures the species once (persistent `Meta.archis`), tracked in a per-zone **Bestiaire** (`showBestiaire` — lists **every** species of each zone incl. boss/miniboss, with capture state on the archi-capable ones; 32 capturable species across the 12 zones; the Kankreblath and Larves-donjon species genuinely have no archi on Metamob — archis attach to the base/outdoor variants). Zone rosters follow the **real DofusDB dungeon compositions** (`api.dofusdb.fr/dungeons/{id}` → `monsters[]` + `bosses[]`) — check there before inventing species. The **Dofus Ocre** is no longer a boss drop — it unlocks in tiers every 50 captures (`OCRE_PALIERS`), granting escalating team passives (PA + damage) via `bonusEquipe(meta)` (combines Dofus + Ocre). Not reachable with the current ~16 monsters by design (framework for a bigger bestiary, ~286).

### Kamas, HDV & Forgemagie

- **Kamas**: per-run currency in `RunState.kamas`, lost on death; earned on victories via `gainKamas` — node type × toile, ±15% variance — and by selling at the HDV. Shown on the map sidebar and in the recap. Constants in `KAMAS` in data.ts.
- **Hôtel de vente**: `hdv` NodeType, tile `cases/hdv.png`, excluded from Incarnam. Stock = `genererStockHDV`, PREMIUM shop: current-toile items only at épique/légendaire, next-toile items from rare up (~40% of slots when it exists; commun/rare local gear comes from fights). Buy price `KAMAS.prix` by rarity × toile, resale 50%.
- **Forgemagie**: `forgemagie` NodeType, weight 6 in `GEN_CARTE.poids`, excluded from Incarnam, tile `cases/forgemagie.png`. The Forgemage upgrades ANY owned instance (inventory or equipped — mutation in place via `forgerInstance`, so the reference stays valid) to its NEXT defined rarity tier; cost = HDV price of the TARGET tier × `KAMAS.forgeCoef` (0.6). The **Forgemage téméraire** costs ×0.3 but fails 30% of the time (kamas lost, item intact — `KAMAS.forgeTemeraire`). Max-tier items (and any instance without a next defined tier) are not forgeable (`rareteSuivante`); `showForgemagie` previews the target tier's stats and records forged tiers in the Armurerie collection.

## Commands

- `npm run dev` — Vite dev server (hot reload) at http://localhost:5173
- `npm test` — run the Vitest suite once. `npm run test:watch` to watch.
- A single test: `npx vitest run src/combat.test.ts -t "règle de ligne"` (`-t` matches the `describe`/`it` name).
- `npm run typecheck` — `tsc --noEmit`, strict mode.
- `npm run build` — typecheck + production bundle to `dist/`.
- `npm run sim` — **headless balancing harness** (`src/sim.ts` via `vitest.sim.config.ts`; kept out of the `*.test.ts` glob so `npm test` ignores it). Replays every encounter N times (IA both camps, seeded RNG) for a reference multi-element team at each zone's expected level, in three gear states (**NU / MI 2-pièces / SET complet**), and prints a win% · turns · HP-left table (flags: `falaise 2→4p`, `stalemate?`…). Tune constants in `data.ts`/`progression.ts`, re-run, compare. **Caveat**: the sim's `controllerIA` heals optimally and never mis-plays → it's a difficulty *floor*; real play is harder (esp. boss fights). See the `sim-harness` memory.
- `node scripts/fetch-assets.mjs` — one-shot fetch of monster sprites from DofusDB (name.fr → gfxId → PNG into `public/assets/monstres/`; skips existing files; direct gfxId overrides for species without an exact DofusDB homonym).
- `node scripts/audit-icons.mjs` — item icon audit, reads `src/content/items.json` (the old `import-items.mjs`/`scripts/items.csv` pipeline is gone; `items.csv` stays as an archive only — see Scope discipline → Item rarity).
- `npm run editor:build` — génère `editeur.html` à la racine (gitignoré), l'éditeur de contenu standalone (HTML+JS inlinés, zéro dépendance) à envoyer au game designer.
- `npm run content:import -- <fichier> [--force] [--sans-assets] [--sans-tests]` — importe un export de l'éditeur : validation 3 passes (schéma, cohérence référentielle, règles métier) + contrôle de fraîcheur par hash (refuse un import basé sur un contenu périmé sauf `--force`), écrit `src/content/*.json`, télécharge les assets DofusDB référencés (sauf `--sans-assets`), puis enchaîne `typecheck` + `test` (sauf `--sans-tests`). Seule porte d'entrée pour modifier le contenu — ne jamais éditer `src/content/*.json` à la main.

## What the game is

A turn-based tactical roguelite in the Dofus universe, played in the browser. The single loop V0 must validate: **tactical combat + a permanent relic (Dofus) that survives death and makes the next run easier**. Everything is judged against one question — *is the combat fun?*

## Tech stack

- **TypeScript + Vite**, vanilla — **no React, no Canvas**. UI is plain DOM. The point is to test *feel*, not visuals.
- **Data-driven**: classes, spells, monsters, encounters, map-generation params and tuning constants live in `src/data.ts`, not hardcoded in logic.
- **Vitest** for the pure modules (`combat`, `progression`, `carte`) — validated headlessly, independent of the DOM.
- Persistence: the `Meta` object (owned Dofus, captured `archis`, `runs`/`victoires` counters, `succes`, `collection`) survives runs, saved to `localStorage` (key `rld_meta_v0`, with rétro-compat defaults in `chargerMeta`). Player *settings* (end-turn key, auto-end-turn, default formation row, per-class preferred element) persist separately under `rld_settings_v0` (`src/config.ts`). The **run in progress** persists under `rld_run_v0` (`sauverRunEnCours`/`chargerRunEnCours`/`effacerRunEnCours` in `run.ts`): saved after every resolved board node and at zone transitions; a combat in progress is NOT saved (refresh = redo the node); cleared on wipe/victory/abandon (abandon counts as a failed run); the start screen offers Reprendre/Abandonner when a save exists. Levels/points/HP live in `RunState` and reset on death.
- Deploy: pushing `main` publishes to GitHub Pages via `.github/workflows/deploy.yml` (`GITHUB_ACTIONS=true` sets Vite `base` to `/RogueLiteDofus/`).

## Code map & architecture

The hard line in this codebase: **`combat.ts` is pure** (no DOM, no `localStorage`) and **`src/ui/` is presentation only** (no game rules). Everything is wired in `main.ts`. Keep new logic on the correct side of that line — it's why the engine is unit-testable.

- `src/types.ts` — the data model (interfaces straight from the spec): combat types, `Progression`, equipment types (`Item` with fixed-stat `tiers`, the rolled-rarity `ItemInstance`, `EquipSlot`), and map types (`MapNode`, `GameMap`).
- `src/data.ts` — all content & tuning: `SORTS` (**12 classes**: Iop, Cra, Eniripsa, Sadida, Sram, Feca, Ecaflip, Ouginak, Roublard, Xélor, Éliotrope, Forgelance + monsters), `CLASSES`, `MONSTRES` (bestiary across 12 zones), `DOFUS` (31-relic catalogue), `COMBATS` (per-zone encounters; boss encounters = **boss + miniboss**), **`ZONES`** (12 zones, flat) + **`TRANCHES`** (level-bracket groupings; t1 = the 12 zone ids in play order — Incarnam → Champs d'Astrub → Tainéla → Tofus → Akadémie des Gobs → Kankreblath → Maison Fantôme → Scarafeuilles → Forgerons → Larves → Grotte Hesque → Nid du Kwakwa, ordered by official dungeon level; t2-t5 empty/inactive; event dungeons Nowel/Halouine/Pwak are reserved and must never enter tranche zone lists; each zone has `pools = {normales, elite, boss}`), `XP_PAR_TYPE`, `TAVERNE_PCT`, `GEN_CARTE` (the board is a **Pokelike diamond lattice**: alternating staggered rows of 2 — columns ±0.5 — and 3 — columns −1/0/+1 — with edges = exact diagonals |Δcol| = 0.5; row count 10-12 rounded to even so the last 2-row converges on the single donjon). Each class/monster has an optional `img` (path under `public/assets/…`). Sprites/icons and base stats/resistances are pulled from **DofusDB** (`api.dofusdb.fr` JSON + `/img/monsters/{gfxId}.png` CDN — keyed by **gfxId**, not monster id; the website itself is a JS SPA and not fetchable). **Archimonstre names** (`archiNom`) come from **Metamob**, which is more complete than DofusDB for archis — see the `archimonstre-source-metamob` memory for how to read its bundled static catalog without an API key.
- `src/content/*.json` — **source de vérité du contenu** (sorts, classes, monstres, combats, zones_pools, items, butin_toiles), sérialisation canonique (`scripts/canonical.mjs` : clés triées, indent 2) pour des diffs git lisibles. `src/data.ts` importe ces JSON et les ré-exporte typés (plus de `src/items_gen.ts`, supprimé). Garde permanente `src/content.test.ts` : le contenu doit passer `scripts/content-validate.mjs` (3 passes) à chaque run de tests.
- `editor/` — sources de l'éditeur de contenu standalone (template HTML + CSS + JS numérotés `00-…`/`60-…`), inlinées par `scripts/editor-build.mjs` dans `editeur.html` à la racine (généré, gitignoré, jamais commité). L'export de l'éditeur ne rentre dans le repo que via `scripts/content-import.mjs`, seule porte d'entrée du contenu (voir Commands).
- `src/combat.ts` — **pure engine**. `runCombat()` is the async turn loop, decoupled from input via **controllers** (`Controller = (acteur, combatants) => Action | null`). The UI passes a promise-based controller for the player; `controllerIA` drives enemies. RNG injected (`hooks.rng`) for deterministic tests. Reads final `c.stats` (via `statsEffectives`, which adds temp stat buffs; it never knows about levels except `c.niveau`, baked for `+x/lvl` scaling). Handles the **support + new-class mechanics** (see below). Dedicated handlers `lancerTarot`/`lancerEspritFelin` for the random Ecaflip spells.
- `src/progression.ts` — **pure**. Levels/XP/points: `xpRequis`, `coutPoint` (escalating 1→2→3 by points already invested), `gagnerXP`, `investir`/`investirN` (bulk allocation, `n = Infinity` for the UI's « Max » button), `restat`, `statsFinales`, `pvMaxFor`, `multOffensif`, `multSoin`, `pctRembPA` (PA-refund chance from Chance, used by Flèche magique). No DOM/storage. `STAT_KEYS` = investable stats (Soin & Prospection are class-only, not investable).
- `src/config.ts` — player settings (`Settings`): end-turn key, `autoFinTour`, default `formation` (classe → grid cell 0-7) and per-class preferred `elements`, persisted to `localStorage` (`rld_settings_v0`) with validation + defaults on load. `nouvelleRun` reads it to seed positions/elements.
- `src/carte.ts` — **pure**. `genererCarte(rng, pools)` builds one zone's node graph by rows with connectivity guarantees (combat ids drawn from the zone's `pools`); `atteignables(carte)` / `noeud(carte,id)` for navigation.
- `src/run.ts` — `RunState`/`PersoState` (per-character progression + current HP + `elementChoisi`, persisted across nodes). **Dynamic roster**: `nouvelleRun(choix)` builds the team from chosen class ids (run starts with **2 of the 11 playable**); `recruter`/`propositionsRecrutement`/`classesHorsEquipe`/`equipePleine` drive Taverne recruitment (team max 4, recruit-at-team-level, swap when full). `EQUIPE_DEPART` is just the default used by tests. Combatant factories **bake final stats** into a `Combatant` (`combattantDepuisPerso`, `equipeCombattante`, `synchroniserPV` writes back HP + chosen element), `soignerEquipe`, `Meta` persistence (`bonusDegatsDofus`).
- `src/ui/` — DOM rendering + player controller, split per screen behind a barrel (`index.ts` = `init()` + full re-export; `main.ts` imports `"./ui"` as before). Layering: `assets.ts` (the `A()` path helper + every icon/menu constant) and `dom.ts` (`root`/`setRoot`, `ecran()`, `escapeHtml`, tooltip plumbing, the SHARED mutable `config`) import no other ui module; `composants.ts` (shared helpers: item chips `itemNomHtml`/`itemStatsHtml`, class cards, stat labels/pct helpers, `renderDofusRack`) imports only those two; `combat.ts` (the whole fight screen: module state, `beginCombat`/`onUpdate`/`render`, FX floats, timeline, spell bar, `playerController`, `log`, keyboard via `initControlesClavier`) and the screen files import assets/dom/composants — screens NEVER import `combat.ts` and vice versa; screen→screen imports are fine. Screens: `accueil.ts` (start/choix équipe/succès/collection Dofus), `carte.ts` (SVG board/zaap/transition), `equipe.ts` (taverne/formation/stat panel/Otomai), `inventaire.ts` (inventaire/drop/settings), `boutique.ts` (HDV/forgemagie), `collections.ts` (bestiaire/armurerie/capture/écran Dofus + `mondeBloc`), `fin.ts` (recap/wipe). Combatant cards show element ronds (clickable to switch strike element), crit/dégât-crit/soin/dgts-finaux mini-stats, all-4 resistances, prospection (allies only). Sprites/icons via the `asset(categorie, id)` convention (`/assets/<cat>/<id>.png`) with `onerror` fallback — drop a file, it shows; absent, it's removed (nothing breaks). Class portraits exist for all classes under `public/assets/classes/`; individual spell icons may still be absent and fall back gracefully.
- `src/main.ts` — orchestration: accueil → for each zone of the **active `TRANCHES` entry** (une run = une tranche; t1 = 12 zones, t2-t5 locked placeholders shown grayed on the start screen), `jouerZone` generates that zone's board and navigates it until the donjon (boss+miniboss) is beaten → next zone; wipe anywhere ends the run. HP/levels persist across zones; only `Meta` survives death.

Two key seams keep things testable and decoupled:
1. **Combatants are rebuilt per fight** from `RunState` — final stats and `pvMax` are computed in `run.ts` and baked into the `Combatant`, so `combat.ts` stays ignorant of levels/points. HP persists via `PersoState.pvActuels` (written back by `synchroniserPV`), not by keeping the combatant object around.
2. **The async controller pattern**: `runCombat` `await`s the active camp's controller, so the same loop runs headless (IA vs IA in tests) or interactively (promise resolved by a DOM click) with no branching in the engine.

### Core combat rules that are easy to get wrong

- **Élément de frappe** (`elementDeFrappe`, `combat.ts`): a character has no fixed element. Element ↔ stat is `terre→force`, `feu→intelligence`, `air→agilité`, `eau→chance`. The strike element is the **player's `elementChoisi` if it is one of their 2 strongest elements, otherwise their single strongest** — so the clickable element ronds let a player pick between their top-two, not any element (exception: `elementLibre` from the Kwakwaffe unlocks all 4). All of a character's damage uses that element, and target resistances are applied per that element.
- **Grid & line rule (SYMMETRIC, 4×2 per camp)**: `position` is a grid cell **0-7** — cells **0-3 = ligne avant**, 4-7 = arrière (`estAvant(c) = position < 4`, `combat.ts`). `ennemi_ligne` spells hit only the target camp's front-row living; if the front row is empty, the back becomes exposed (`ligneFront`). Applies to **both** camps. `ennemi_tous` ignores it; rebound (Flèche perçante) hops by ascending position; taunt (`provoque`) overrides. Placement is free: all-front, all-back, mono-tank, etc. Players placed via **formation** (`config.formation` = `{classeId: cell}` → `PersoState.position`), edited from the board grid (`showFormation`); enemies placed by encounter `position` cells in `data.ts` (up to 8). Combat cards are vertical, grouped into front/back rows per camp.
- **Damage pipeline** (order matters, see `degatsCible` in `combat.ts`): esquive check (Agilité, capped) → roll (or max if `maxRoll` charge) → add élément-de-frappe stat × scaling → crit (Force, capped, *adds* damage via Agilité, does not double) → `degatsInfliges` debuff → rebound mult → resistance (unless `ignoreResistances`) → Dofus team bonus (`ctx.playerDamageBonus`, player camp only) → `multOffensif` (Intelligence, capped, all casters) → round, floor at 0.
- **Support mechanics** (Eniripsa kit, all in `combat.ts`): damage goes through `infligerDegats` → **shield (`bouclier`) absorbs before HP**; `poison`/`hot` are timed effects in `effets[]` that **tick at the start of the actor's turn** in `effetsDebutTour` (poison can kill, then transmits to the combatant `derriere` if `transmet`); heals via `soigner` (incl. `soinComplet`, `allie_tous`, `soinEquipeRatio` lifesteal-to-team); `appliquerSoutien` applies bouclier/HoT/`dissipe`/`paGain`/`bonusProchainSortPct`; per-target **cooldowns** live on `acteur.cooldowns` and are enforced inside `ciblesValides`. `allie_tous`/`soi`/`invocation` spells resolve with no target click.
- **Invocation & init (Sadida kit)**: the turn loop re-picks the next actor each step by **effective initiative** (`initOf` = base + `initiative` effects), so a mid-round init debuff (Déferlante/Étreinte) actually delays enemies that haven't acted yet. **Invocations** (`estInvocation`, e.g. the Poupée de garde via `invoquerPoupee`) occupy a slot, are skipped in the loop, and can `provoque` — enforced in `ciblesValides` (enemies must target a provoking ally first). One poupée per summoner. `Vigueur des bois` sets `bonusOffensifProchain`, a one-shot % consumed by the next damage cast.
- **Secondary-stat caps are mandatory**: crit, esquive, puissance offensive are capped (~50%) in the formulas. Don't remove the `Math.min` caps.
- **HP persists between fights/nodes** within a run — the Taverne node is the only heal. A wipe ends the run but **must not erase `Meta.dofus`**.
- **PV model**: `pvMax = pvBase + vitalitéFinale × PV_PAR_VITA` (spec-literal). Monster damage in `data.ts` is tuned against the resulting (inflated) player HP — retune both together.

## Scope discipline

Each tranche is deliberately narrow; the spec's validation criteria define "done". Everything in the "out of scope" list above is **deferred** — resist pulling it in. Values in `data.ts`/`progression.ts` are meant to be adjusted. Crit is a multiplicative bonus (affects players AND monsters).

### Current build

**11 playable classes** (Iop/Cra/Eniripsa/Sram/Feca/Ecaflip/Ouginak/Roublard/Xélor/Éliotrope/Forgelance; Sadida disabled pending rework), **4 elements**, in-run levels/points, recruitment (start at 2, taverne recruit/heal, max 4), equipment (4 slots, per-run loot), **Tranche 1 = 12 sequential zones** ordered by official dungeon level (see `TRANCHES`), a 31-relic collection, and a start screen showing the 5 tranches (t2-t5 locked).

### Mode Ascension

Once a tranche has been cleared once, it becomes **replayable at an escalating difficulty**: 8 cumulative paliers A1→A8, catalogue in `ASCENSION` (`data.ts`), merged via `effetsAscension(palier)` in `run.ts` (booleans OR, percentages "last wins", multipliers compose, bonuses add — see `ascension.test.ts`). Effects: wider normal-fight packs (`packPlus1`, +1 zone monster, never in the donjon), reduced taverne heal (`tavernePctAscension`, 50%→30% from A2), enraged bosses (`bossEnrage`, stacking per-turn damage via `appliquerEnrage` in `combat.ts`), global enemy PV/offensive-stat multipliers, doubled elite modifiers from A5 (`appliquerModificateursElite(..., 2)`, `genererCarte` draws 2 distinct ids per combat_dur node), reduced starting/recruit HP (`pvDepartPct`), and a final-zone-only PA bonus on the last boss. Applied to enemy packs via `appliquerAscensionEnnemis` (`run.ts`), keyed by encounter `type` (`combat`/`combat_dur`/`donjon`) and `derniereZone`. **Per-tranche record** in `Meta.ascension` (`recordAscension`/`enregistrerAscension`, never decreases); the Dofus drop rate gets **+1%/palier** (`tauxDofusAscension`). Palier picked on a start-screen selector (locked above the tranche's current record + 1). Four achievements gate on it (`ascension_1/3/5/8` in `SUCCES`, run.ts). `npm run sim` measures it too via a top-of-file `ASCENSION` constant (default 0 = base game; taverne/starting-HP effects are out of the sim's scope since it only measures isolated encounters, not full runs — documented in a comment next to the constant) — A0 reproduces the reference win-rate table below; A8 degrades progressively (no early-zone brutal 0%, later zone bosses trend toward unwinnable by design).

### Boss signatures

**All 12 bosses have a signature mechanic** on top of the shared `ecrasement/charge/morsure` kit. Damage-spell signatures: a 6-PA spell with a rider, listed FIRST in `sorts` (aggressive IA plays highest-cost first, ties broken by list order) and gated by `cooldownTours`. Engine-level signatures: Kwakwa's `mueElementaire` (each of its turns: 65% resist everywhere except one random element at 0 — `appliquerMueElementaire`), Grunob's `bonusParAllieLigne` passive (+15% damage per living same-row ally, folded into the damage mult in `lancerSort`), and invocation-type signatures played by `iaAgressif` in priority when `invocationUtile` (Kankreblath `invoqueMonstre` summons a zone monster — summons are FULL combatants that play their turns, unlike the Sadida Poupée; Shin Larve's 2-PA ponte lets her summon AND attack; Boostache `ressuscite` revives a fallen non-boss ally). Tests in `boss.test.ts`. Sim note: summoned actors inflate the sim's per-actor turn counts on those boss rows.

**Boss rooms are 4v3/4v4 and bosses have 10 PA** (signature 6 PA + attack 4 PA the same turn; boss offensive stats were scaled ~×0.6 to compensate the doubled action rate; rooms add 1-2 zone species incl. back-row shooters — designed after playtest feedback that 2-enemy boss rooms were too simple).
